# 二、作业脚手架（homework-scaffolder）

## 它解决什么问题

以前你每新开一门课，都要手动建目录、写 CLAUDE.md、填课程信息、写共享知识引用。现在一句话搞定。

## 怎么用

### 基础用法

```
新建作业 数据库原理 MySQL
```

或者更具体的：

```
新建作业 UE5高级开发 "C++/蓝图" "Unreal Engine 5.7"
```

### 它会在你的电脑上创建

```
/Path/To/Your/GlobalConfig/YourProject\YourCourseName\
├── CLAUDE.md              ← 课程信息 + 共享知识引用
├── .claude\
│   └── memory\
│       └── MEMORY.md      ← 记忆索引
└── Photos\                ← 空文件夹（给你放截图）
```

### 安全机制

- 如果目录里已经有 `CLAUDE.md`，**拒绝创建**，不会覆盖你写好的东西
- 只会在 `/Path/To/Your/GlobalConfig/YourProject\` 里面建东西，不会碰别的目录

# 三、UE5 蓝图审计器（ue5-blueprint-auditor）

## 它解决什么问题

你 UE5 项目里有 ~90 个蓝图，命名有没有遵循规范？哪些漏了前缀？这个工具帮你扫一遍。

## 怎么用

### 查看所有命名规范

```
帮我看看 UE5 有哪些命名前缀规范
```

会列出 20 种前缀，比如：
- `BP_` 开头 → 蓝图类
- `BPI_` 开头 → 蓝图接口
- `ABP_` 开头 → 动画蓝图
- `IA_` 开头 → 输入动作
- ...等等

### 扫描一个项目

```
审计一下 Demo02 的蓝图命名
```

### 它会告诉你

- 一共扫了多少个文件
- 多少个命名规范 ✅
- 多少个命名不规范 ❌（包括文件名、在哪个文件夹、建议改成什么前缀）
- 关卡文件（`.umap`）允许中文命名，不会误报

## 安全机制

- **绝对不读 .uasset 文件的内容**，只看文件名
- **绝对不修改任何文件**，只输出报告

# 四、Obsidian 笔记管理器（obsidian-vault-manager）

## 它解决什么问题

你 UE5 学习过程中积累了大量知识点（Chaos 物理、BPI 通信、摄像机架构...），但散落在对话里没有归档。这个工具帮你自动写入 Obsidian 笔记库，而且不会搞乱已有的 17 篇笔记。

## 你的笔记库在哪里

```
/Path/To/Your/UE_Project/UENoteBook\
```

里面有 17 篇结构化笔记（01 到 17），每篇都有严格的标题层级和标记符号。

## 怎么用

### 查看笔记库状态

```
看看我的 UE5 笔记库有多少篇
```

输出: `17 notes, 111.1 KB`

### 保存对话中的知识点

```
把刚才讨论的 Chaos 物理双端驱动存到笔记
```

执行流程：
1. 先用 **dry-run（预览模式）** 展示要插入什么、插在哪里
2. 你确认没问题 → 加 `--write` 真正写入
3. 写入时会找笔记中最匹配的标题位置，而不是往末尾硬塞

### 检查有没有"孤岛笔记"

```
检查一下有没有孤立笔记
```

找出没被其他笔记引用的笔记，方便你建立链接。

### 画笔记关系图

```
把笔记之间的关系画出来
```

输出 Mermaid 图，展示哪些笔记引用了哪些笔记。

如果 17 篇笔记之间还没有互相引用链接，会提示"当前笔记库连接稀疏"，而不是输出空图。

## 安全机制

- **默认不写入**，必须加 `--write` 才真正写
- **结构感知插入**：会先分析笔记的标题层级，找到最相关的位置再插入
- **写入前展示差异对比**：让你看到会在哪里加什么内容
- **不覆盖已有内容**：只追加，不修改原文

# 五、10 条竞赛级工程铁律

## 它从哪里来

从百度 CTI 推理优化竞赛 38 次提交的试错中提取。

## 写在哪里

```
/Path/To/Your/GlobalConfig/.claude\shared\project_rules.md
```

## 以后怎么生效

每次用作业脚手架新建课程，新 CLAUDE.md 会自动引用这个文件。我会在写代码时自动遵循这些规则。

## 10 条铁律速查

| # | 规则 | 一句话解释 |
|:--:|------|------|
| 1 | **单变量提交** | 一次只改一个变量，不然不知道谁在起作用 |
| 2 | **改前备份** | 改代码前先复制一份，用备份覆盖不用回退 |
| 3 | **理论≠实测** | AI 说对的，实际可能翻车，必须实测验证 |
| 4 | **提交前检查清单** | 语法 / 压缩包内容 / 注释头 / 备份，缺一不可 |
| 5 | **三方推理强制** | Claude + QwenMax + Gemini 三方必须交叉验证 |
| 6 | **回退≠恢复** | 反复改完再 git 回退可能有残留差异，必须硬覆盖 |
| 7 | **警惕无 GPU 盲狙** | 没显卡跑过的代码不要直接提交 |
| 8 | **buffer 精度陷阱** | PyTorch 转换半精度时 buffer 不受影响，需单独处理 |
| 9 | **隔离外网依赖** | 无网评测环境不能有 pip install 命令 |
| 10 | **容忍算力波动** | GPU 跑分 ±1-2 秒是正常的，别过度解读 |
